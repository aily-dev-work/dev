# いまやること

Render にデプロイしたあと、**この順番**で進めてください。

---

## 1. Render にまだデプロイしていない

→ **[DOCKER_DEPLOY.md](DOCKER_DEPLOY.md)** の「2. Render でデプロイ」に従い、Web サービスを作成し、環境変数を入れてデプロイする。

---

## 2. 5 分ごとにジョブを回したい

→ **[RENDER_CRON_AND_DB.md](RENDER_CRON_AND_DB.md)#時系列でやること一覧** の **「5 分ジョブ（cron-job.org）」** の表の **#1 から #11** を順にやる。

（cron-job.org を開く → サインアップ → ジョブ作成 → URL・POST・X-Cron-Secret・5 分間隔 → 保存 → Run now で確認）

---

## 3. データを消したくない（DB 永続化）

→ **[SUPABASE_SETUP_SIMPLE.md](SUPABASE_SETUP_SIMPLE.md)#時系列でやること一覧** の表の **#1 から #31** を順にやる。

（Supabase でプロジェクト作成 → 接続文字列をメモ → Render に DATABASE_URL を追加 → ローカルで migrate → エラー時は表の #29〜#31 の対処）

---

## 4. フロントを Vercel にデプロイしたい

→ **[VERCEL_FRONTEND.md](VERCEL_FRONTEND.md)** の **時系列でやること一覧** に従う。

（Vercel でプロジェクト作成 → Root Directory を `frontend` に → 環境変数 `NEXT_PUBLIC_API_BASE_URL` に Render の URL → デプロイ → Render の **CORS_ORIGINS** に Vercel の URL を追加）

---

## まとめ

| やりたいこと           | いまやること |
|------------------------|--------------|

| 5 分ジョブを動かす     | 2 → RENDER_CRON_AND_DB の時系列 #1〜#11 |
| DB を残す（Supabase）  | 3 → SUPABASE_SETUP_SIMPLE の時系列 #1〜#31 |
| フロントを Vercel に載せる | 4 → VERCEL_FRONTEND.md |
