# 長期トレンド考慮・詳細分析の設計メモ

## 狙い

- **長期トレンドも考慮した多時間軸の見方**  
  例: 長期は上昇トレンド、短期は下降、現在のシグナルは様子見、といった状態を一覧できるようにする。
- **詳細な分析文**  
  例: 「〇円まで上がれば〇円まで上昇見込み」「〇円まで下がれば〇円まで下がるおそれあり」のように、価格シナリオ付きのコメントを出せるようにする。

## 想定する出力イメージ

- **トレンド要約**
  - 長期トレンド: 上昇 / 中立 / 下降
  - 短期トレンド: 上昇 / 中立 / 下降
  - 現在の判定: 買い / 売り / 様子見
- **文章化された分析（例）**
  - 「長期は上昇、短期は下降。現在は様子見。」
  - 「〇〇円まで戻せば〇〇円まで上昇見込み。〇〇円を下回れば〇〇円まで下落リスク。」
- **根拠となる数値**
  - 長期・短期で参照している期間（例: 長期=75日、短期=25日）
  - 抵抗・支持とみなす価格帯（高値・安値、MA、など）

## 必要な拡張

### 1. 多時間軸のトレンド判定

- **現状**: 日足で 1 本のテクニカルサマリ（trend_long / trend_mid / trend_short）と、それから出る「買い・売り・様子見」が 1 つだけある。
- **拡張案**:
  - **長期トレンド**: 長い期間（例: 75日 or 週足）の終値・MA・高安から「上昇 / 中立 / 下降」を算出。
  - **短期トレンド**: 短い期間（例: 25日 or 5日）で同様に算出。
  - **現在のシグナル**: 既存のスコア（買い・売り・様子見）をそのまま「現在の判定」として使う。
- 実装イメージ: 日足に加え、必要なら週足や「長期用・短期用」の別ウィンドウでテクニカルを計算し、トレンドラベルを返す API/サービスを用意する。

### 2. 価格シナリオ（〇円まで〜）

- **抵抗・支持の算出**
  - 直近高値・安値、MA、ボリバン帯、などから「ここを超えると上」「ここを割ると下」の価格帯を計算。
- **シナリオ文の生成**
  - 「〇〇円（抵抗）まで上がれば〇〇円まで上昇見込み」「〇〇円（支持）を下回れば〇〇円まで下落のおそれ」のような文言をルールベース or AI で生成。
- **出力形式**
  - 構造化データ（抵抗価格、支持価格、上側ターゲット、下側ターゲット）＋ 短文の「分析コメント」を API で返し、フロントで表示。

### 3. 表示・API

- **銘柄詳細 or ダッシュボード**
  - トレンド要約（長期・短期・現在）と、価格シナリオ付きの「詳細分析」を 1 ブロックで表示。
- **API**
  - 例: `GET /api/v1/stocks/<id>/analysis/` で「多時間軸トレンド ＋ 価格シナリオ ＋ 分析文」を返す。  
  - 既存のスコア API を拡張するか、分析専用エンドポイントを新設するかは検討。

### 4. トレードスタイルとの関係

- **長期プロファイル**: 長期トレンドを重視し、シナリオも長期の支持・抵抗を多めに使う。
- **短期プロファイル**: 短期トレンドと直近の支持・抵抗を中心に説明。
- **デイトレ**: 5 分足ベースの短期トレンドと、当日の値幅・押し安値・戻り高値などを用いたシナリオ。

## 補足

- 詳細分析は **シグナル発報とは独立**してもよい。  
  まず「今の長期・短期・様子見と、〇円まで〜のシナリオ」を**参照用**で表示し、発報条件は既存の閾値ベースのままにしておく運用も可能。
- 分析文の自動生成には、ルールベースの他に **AI** を組み合わせる選択肢もある（プロファイルの重み・閾値と多時間軸の数値を渡して短文を生成するなど）。
- **AI の分析材料**: 長期トレンド・短期トレンド・現在の判定（bias）は `TechnicalSummary` および GET /api/v1/stocks/scores/ と GET /api/v1/stocks/<id>/score/ の `technical_summary` で取得可能。将来的に analysis_package や AI 改善提案の入力にこれらのフィールドを含めると、トレンドを考慮した提案がしやすくなる。

## 解釈・コメントは AI に任せる方針

- **やりたいことの例**
  - 長期トレンドは上昇だが短期は下降 → 「〇円付近で押し目買いを検討」「〇円を割ると長期上昇トレンド終了の可能性」
  - 価格と支持・抵抗を踏まえた「〇円で買うべき」「〇円だと長期上昇トレンドが終わりかも」といった文章化された解釈。
- **方針**: 上記のような**文脈を踏まえた解釈・コメントは、AI 実装後に AI に判定・生成させる**。
  - 現段階では、長期・短期トレンドと現在の判定を**データとして表示**し、抵抗・支持やシナリオ文はルールベースで出すかは別タスク。
  - AI 実装後は、多時間軸トレンド・価格・支持抵抗・プロファイル情報などを入力に、「〇円で買うべき」「〇円を下回るとトレンド転換の可能性」などの**短文コメントを AI に生成**させ、ダッシュボードや銘柄詳細に表示する形を想定する。
- ルールで網羅するにはパターンが多く複雑になるため、**解釈は AI 任せ、データ（トレンド・価格・スコア）は従来どおり算出**する役割分担にするとよい。
