# プロファイル成績指標の意味と継続性（設計メモ）

## 平均リターンとは

- **定義**: 各シグナルについて `return = (評価時点の価格 - 基準価格) / 基準価格` を計算し、その平均を取ったもの。
- **意味**: 基準価格を「掛け金」とみなしたとき、**何%増減したか**（率）に相当する。例: return 0.02 → 2% の値上がり、-0.01 → 1% の値下がり。手数料・スリッページを除いた価格ベースの率。
- レバー1倍・全額投入なら「掛け金が何%になったか」の平均に一致する。レバーや部分投入はここでは考慮していない。

## 成功率・リターンは「ワンショット」ではなく継続性を見たい

- 現状: 集計期間内の全シグナルを一括で集計し、**1 本の成功率・平均リターン**を出している。
- 課題: 「今回だけいい成績だったから優秀」と判断してしまうと、たまたまの当たりや短期の過学習でプロファイルを過大評価しがち。
- 望ましい姿: **継続的に**良い成績が出ているかを確認できるようにする。

### 継続的な成績の見方（案）

1. **期間を分けて比較**
   - 例: 直近 3 ヶ月の成功率・平均リターン vs その前 3 ヶ月。両方とも良い／悪化していない、なら「持続的」とみなす。
   - ダッシュボードや API で「直近」「前期」の 2 本立てで表示する。

2. **ロールングウィンドウ**
   - 例: 直近 30 件（または 50 件）のシグナルだけを対象にした成功率・平均リターンを常に表示。時系列で「最近の成績」の推移が見える。
   - または「直近 N 件」をスライドさせて、N 件あたりの成功率のばらつきを確認する。

3. **最低サンプル数と信頼区間**
   - 評価件数が少ないうちは「優秀」と表示しない／グレー表示する。例: evaluated_count &lt; 20 は参考値とする。
   - （将来）成功率の信頼区間を出し、「下限が閾値を上回っているか」で判断する。
